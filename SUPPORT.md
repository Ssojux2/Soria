# Support

Soria is early open-source software. The fastest way to get useful help is to
choose the right channel and include safe, reproducible details.

## Where to Ask

- Use GitHub Issues for reproducible bugs.
- Use GitHub Discussions for setup help, ideas, DJ workflow questions, and
  compatibility notes.
- Use private vulnerability reporting for security issues.

## Before Opening an Issue

- Check the README and `docs/` folder.
- Confirm whether you are using a release DMG/ZIP or a source build.
- Validate the analysis worker from Settings.
- Try a small synthetic or disposable audio sample before reporting
  normalization or export behavior.

## Safe Details to Include

- macOS version and Mac architecture.
- Soria version or commit SHA.
- Whether the app came from DMG, ZIP, or source.
- Redacted error messages.
- The export/import format involved, such as Rekordbox XML or Serato CSV.

## Do Not Include

- API keys.
- Real music files.
- Private DJ library databases.
- Unredacted home directories, music paths, screenshots, or logs.

Early DMG/ZIP builds are not notarized. If macOS blocks the app and you trust
the source, use System Settings > Privacy & Security > Open Anyway, or build the
app yourself with Xcode.
