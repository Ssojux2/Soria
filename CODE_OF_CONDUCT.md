# Code of Conduct

Soria should be a calm, practical place to collaborate on music software.
Contributors, users, maintainers, and visitors are expected to help keep the
project respectful and useful.

## Expected Behavior

- Be considerate, patient, and specific.
- Assume good intent while still naming concrete problems.
- Keep discussions focused on Soria, DJ workflows, audio analysis, packaging,
  privacy, and project maintenance.
- Respect different experience levels, languages, music scenes, and toolchains.
- Give maintainers enough context to reproduce and evaluate a report.

## Unacceptable Behavior

- Harassment, threats, personal attacks, or discriminatory language.
- Sexualized language or attention in project spaces.
- Publishing private information, private music-library details, API keys, or
  personally identifying logs without consent.
- Repeated off-topic posting, spam, or pressure for unpaid support.
- Misrepresenting unofficial builds, forks, or packages as official Soria
  releases.

## Enforcement

Maintainers may edit, hide, lock, or delete comments; close issues or pull
requests; block users; or take other reasonable moderation action when behavior
harms the project or its participants.

If you need to report conduct concerns, contact the maintainer through GitHub.
If the report includes a security issue or private user data, use the security
reporting path in [SECURITY.md](SECURITY.md).
